HTML

  • The head element contains information, or metadata, about the webpage.
  • The body element represents the visible content shown to the user.
  • Semantic HTML elements have names that describe the content/role they play on a webpage.
  • The !DOCTYPE declaration element tells the browser what document type it should expect when it is opened.
  • The html element represents the root, or start, of an HTML document.
  • An HTML attribute defines additional information about an element (always in opening tag).
  • The title element defines the title (name listed on tab) for a webpage and is important for SEO.
  • The style element defines the CSS styles associated with a website.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • Parent elements hold all the children elements.
  • Child elements are nested inside parent elements.
  • Div elements create a division in the page.
  • Span elements create an inline division.
  • H1-h6 elements represent the level of heading a given text block represents.
  • The p element represents are paragraph or block of text.
  • The ul element represents an unordered (bulleted) list.
  • The ol element represents an ordered list.
  • The li elements represent list items.
  • The img element contains information about images that are display on the webpage.
  • The src attribute defines the location, or source, of the image file.
  • The alt attribute contains a text string that describes the appearance and functionality of the image.
  • The br element creates a line of empty space, or a line break, between two blocks of content.
  • The a element (anchor) creates links to the same webpage or other webpages.
  • The href attribute points to the URL for the link in the anchor element.
  • The main element holds the main content of the webpage.
  • The footer element appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The box model says that each HTML is a box comprised of four layers (content, padding, border, margin).
  • A selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain CSS property (i.e. color) and CSS value (i.e. blue)
  • The "*" wild card symbol is used as a selector to apply style to ALL elements of the page.
  • Selectors can be combined with a "," to allow the declaration to affect multiple selectors.
  • The text-align property allows you to center a title or heading on a webpage.
  • A link element must be added to the HTML document in order to apply styles from an external CSS.
  • A class attribute allows you to add style to specific elements via class selectors.
  • THe box-shadow property adds shadow effects around an element's frame.

Git

  • Git: Version control system that lets you manage and track each version of your project locally. Accessed via the command line.
  • GitHub: cloud-based hosting service that allows us to keep track of our projects inside repos that are stored remotely.
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: adds all changes to file in working directory to the Git staging area.
  • git commit -m "message": commits staged changes to repo.
  • git pull origin main: pulls the base branch into your feature branch. Do this before pushing up any code.
  • git push origin branch-name: pushes feature branch changes to the remote repo (Github)

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script. Javascript is read top to bottom in order.
  • Each line of Javascript code ends with a semicolon.
  • A string is a sequence of one or more characters enclosed in quotation marks that may consist of letters, numbers, or symbols.
  • The console.log method outputs a message to the web console by adding an argument.
  • While a CSS stylesheet (style.css) must be linked within the head element, a Javascript file (script.js) must be linked within the body element.
  • Chrome DevTools allows you quickly test and debug changes made to Javascript by viewing console.log output.
  • You can open Chrome DevTools by right-clicking anywhere on the browser window and selecting "Inspect." Then click the Console tab to see the web console.
  • An if statement is a way of introducing decision-making in our code. When in use, the code will only execute if the statement meets a condition, or is true.
  • If a condition is truthy, it will complete the action requested in between the curly brackets.
  • If a condition is falsy, it will ignore the action in between the curly brackets.
  • The strict equality operator (===) checks to see if two values are equal and returns a Boolean result of "true" if equal and "false" if not equal.
  • An array is a single variable that is used to hold a group of data that is typically related. Can contain strings, numbers, and Boolean values.
  • Using each piece of data's index, or location number, we can access individual data inside an array. JavaScript is zero-indexed and sequential.
  • A for loop used the predictable pattern of indices to perform a task on all items in an array by allowing a single code block to be executed repeatedly.
  • The first statement of a for loop determines the starting point of the loop (var x = 0).
  • The second statement of a for loop is the condition (x < varName.length).
  • The third statement allows the array to iterate (x++ is shortcut of writing x+1).
  • Commenting out code makes it inactive without deleting it, can be referred to later. Put /* before and */ after code.
  • A function is a set of instructions that tells the computer how to perform a certain task.
  • To declare a function, we use the function keyword and then give our function a unique name followed by parentheses. Then add curly brackets that contain code.
  • To execute a function, we must call it in our code.
